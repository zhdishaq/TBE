# Quick Task 260424-20h: Share infra across services

## Goal

All 11 .NET services + gateway can boot via `docker compose up` alone. No per-service launch steps. Keycloak at `https://openid.3ha.one`, YARP gateway at `https://booking.3ha.one`. Infra endpoints (Redis, RabbitMQ, MSSQL) and secrets (Stripe, Keycloak client secrets, GDS creds) live in `docker-compose.yml` environment blocks sourced from `.env`.

## Locked conventions

- **Keycloak Authority (per service)** — `https://openid.3ha.one/realms/{realm}` with realm selected by the service's primary consumer (B2C for customer-facing services, per existing pattern already in BookingService/PaymentService/FlightConnector/Pricing/Notification).
- **Keycloak Audience** — `tbe-b2c-api` (matches Gateway scheme audience; leave unchanged).
- **Gateway Keycloak BaseUrl** — `https://openid.3ha.one` (was `http://keycloak:8080` / `http://localhost:8080`). The gateway fetches JWKS from `{BaseUrl}/realms/{realm}/protocol/openid-connect/certs`.
- **KeycloakB2B BaseUrl** (PaymentService + BookingService) — `https://openid.3ha.one` (was `http://localhost:8080`).
- **Gateway public URL** — `https://booking.3ha.one` (already in docker-compose `backoffice-web` NEXTAUTH_URL).
- **Intra-docker service names unchanged** — `booking-service:8080`, `payment-service:8080` etc. stay as-is in Gateway ReverseProxy.Clusters (internal resolution); only the external Keycloak + public gateway URL change.
- **Runtime secrets via env** — docker-compose passes `ConnectionStrings__*`, `Stripe__*`, `KeycloakB2B__ClientSecret`, `Encryption__ActiveKeyBase64`, `SendGrid__ApiKey`, GDS creds, OpenTelemetry endpoint via `${VAR}` substitution from `.env`. Service appsettings.json hold development defaults only; `.env` overrides in prod.

## Must-haves

1. Every service appsettings.json has `Keycloak.Authority = https://openid.3ha.one/realms/tbe-b2c` (or the service's correct realm).
2. PaymentService + BookingService `KeycloakB2B.BaseUrl = https://openid.3ha.one`.
3. Gateway `Keycloak.BaseUrl = https://openid.3ha.one` in both `appsettings.json` and `appsettings.Development.json`.
4. Every service appsettings.json has `RabbitMQ.Host = rabbitmq` (docker name), `Redis.ConnectionString = redis:6378,password=...` placeholder, and `ConnectionStrings.{Service}Db` placeholder.
5. `docker-compose.yml` passes all runtime secrets via `environment:` blocks: `ConnectionStrings__*Db`, `Keycloak__Authority`, `Keycloak__Audience`, `KeycloakB2B__*`, `Stripe__*`, `Encryption__*`, `SendGrid__ApiKey`, `OpenTelemetry__OtlpEndpoint`, `Amadeus__*`, `Sabre__*`, `Hotelbeds__*`, `Wallet__*`.
6. Every service block depends on `rabbitmq` + `redis` healthy (already true); Keycloak container commented out since it's external.
7. `.env.example` lists every new env var (add KEYCLOAK__AUTHORITY__*, Gateway public URL, any missing ones).
8. SearchService appsettings modifications in git status preserved (external FlightConnector/Pricing base URLs, Redis).
9. FlightConnectorService controller change in git status preserved (the `[Authorize]` addition — not part of this task).
