# Quick Task 260424-2d9: Complete OTel wiring ‚Äî SUMMARY

## Outcome

`AddTbeOpenTelemetry(builder.Configuration, "<ServiceName>")` now runs in all 10 runtime entry points (9 services + gateway). Every service emits OTLP traces + metrics through the shared `SensitiveAttributeProcessor` pipeline (COMP-05 / COMP-06) to the endpoint from `OpenTelemetry:OtlpEndpoint`.

## Findings that changed scope

Initial claim from the dispatcher was "no service's Program.cs calls the helper" ‚Äî not true. **5 of 10 Program.cs files were already wired** in earlier phases (BookingService, PaymentService, FlightConnectorService, PricingService, NotificationService). Only 5 files actually needed the call added.

## Why this placement

Placing the call right after the Serilog host configuration and before other registrations guarantees the tracer provider is registered before any `AddHttpClient`, `AddDbContext`, or `AddMassTransit*` call ‚Äî which matters because those registrations rely on OTel instrumentation (auto-tracing of HTTP/EF/MassTransit) being in place at container build time.

## Decisions

- **serviceName = PascalCase project short name** ‚Äî matches the `Enrich.WithProperty("Service", "‚Ä¶")` string already used in each file and matches the convention set by the 5 previously-wired services. The docker-compose service names (`booking-service`, `payment-service`, etc.) are intentionally NOT used for OTel resource naming; those are container DNS names, not logical service identities.
- **Gateway serviceName = `"TBE.Gateway"`** ‚Äî matches its own Serilog enrichment string.
- **No csproj changes** ‚Äî every target already had `<ProjectReference Include="...\TBE.Common.csproj" />`; verified via grep.
- **No helper changes** ‚Äî `TelemetryServiceExtensions.cs` and `SensitiveAttributeProcessor.cs` untouched per scope boundary.

## Next steps (out of scope, but worth flagging)

1. **Deploy an OTLP collector** ‚Äî `OpenTelemetry:OtlpEndpoint` defaults to `http://localhost:4317`, which means traces hit the void inside docker. Add a `otel-collector` container (Grafana Agent / Tempo / Jaeger / Alloy) to `docker-compose.yml`, or set `OPENTELEMETRY__OTLPENDPOINT=http://otel-collector:4317` in `.env`.
2. **MassTransit tracing is registered** (`.AddSource("MassTransit")`) ‚Äî traces span-link RabbitMQ publishes to consumers, but requires a running collector to observe.
3. **Logs-via-OTLP** ‚Äî current wiring covers tracing + metrics only. Logs still go through Serilog to the console. A future follow-up could add `OpenTelemetry.Logs` + `AddOtlpExporter` on `builder.Logging` to unify the pipeline.
