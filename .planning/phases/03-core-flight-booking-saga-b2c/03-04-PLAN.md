<objective>
Deliver NOTF-01, NOTF-03, NOTF-04, NOTF-05, NOTF-06: a NotificationService that consumes booking lifecycle events from RabbitMQ and dispatches branded transactional emails (with PDF attachments where applicable) within the 60-second SLA, exactly once per (event, email-type) pair.

**Revision-2 scope clarifications:**
- **NOTF-02 (hotel voucher email) is NOT in this plan** — relocated to Phase 4 per ROADMAP update (B2).
- Canonical NOTF ID mapping re-aligned per B3 (see scope notes above). Every consumer ↔ requirement mapping in this plan uses the canonical mapping; do not assume older drafts were correct.
- Task 1 split per W2 into Task 1a (templates + RazorLight renderer + QuestPDF generator + template models) and Task 1b (EmailIdempotencyLog + NotificationDbContext + infrastructure abstractions + SendGrid delivery) to keep each task under the context-cost budget.

Concretely:
1. **Task 1a — Template + rendering + PDF infrastructure (NOTF-06 branded templates, NOTF-01 PDF)**: `IEmailTemplateRenderer` + `RazorLightEmailTemplateRenderer` loading cshtml templates from `Templates/` with typed view models; `IETicketPdfGenerator` + `QuestPdfETicketGenerator`; 5 cshtml templates + 2 partials + 5 typed models; template/PDF unit tests.
2. **Task 1b — Delivery + idempotency + DbContext (NOTF-01 delivery, NOTF-06 idempotency)**: `IEmailDelivery` + `SendGridEmailDelivery`; `NotificationDbContext` + `EmailIdempotencyLog` + migration `20260418000000_AddNotificationTables.cs`; SendGrid + idempotency unit/integration tests.
3. **Task 2 — Consumers + Program.cs wiring + Worker.cs removal (NOTF-01, NOTF-03, NOTF-04, NOTF-05 consumer chain)**: 6 consumers (`BookingConfirmedConsumer`, `BookingCancelledConsumer`, `TicketIssuedConsumer`, `BookingExpiredConsumer`, `TicketingDeadlineApproachingConsumer`, `WalletLowBalanceConsumer`); Program.cs rewired via `AddTbeMassTransitWithRabbitMq`; `Worker.cs` deleted; consumer + idempotency integration tests.

Note: `TicketIssuedConsumer` and `BookingConfirmedConsumer` together deliver NOTF-01. The e-ticket number comes from `TicketIssued`; the customer-facing confirmation email is sent on `BookingConfirmed`. The e-ticket PDF attachment is included with the confirmation email (the consumer reads the e-ticket number off BookingConfirmed and renders the PDF via `IETicketPdfGenerator`). `BookingExpiredConsumer` is a supporting consumer for the BookingExpired event emitted by the saga in 03-01; it has no direct requirement but is required for operational completeness (sends the customer an "unfortunately your booking expired" email). It is covered under NOTF-03's cancellation-family of lifecycle notifications.

Purpose: This plan runs in **Wave 2 in parallel with 03-02** because it shares zero `files_modified` with the payment plan and depends only on the saga contracts published by 03-01 (`BookingConfirmed`, `BookingCancelled`, `TicketIssued`, `BookingExpired` events). It cannot run in Wave 1 because the contracts must exist first.

Output: ~30 new files in NotificationService, 5 unit test classes, removal of Worker.cs placeholder, and a new EF migration adding `EmailIdempotencyLog` table.
</objective>

    using TBE.Contracts.Events;
    using TBE.NotificationService.Application.Email;
    using TBE.NotificationService.Application.Pdf;
    using TBE.NotificationService.Infrastructure.Persistence;

    public sealed class BookingConfirmedConsumer(
        NotificationDbContext db,
        IBookingContactClient bookings,
        IEmailTemplateRenderer renderer,
        IETicketPdfGenerator pdfGen,
        IEmailDelivery delivery,
        ILogger<BookingConfirmedConsumer> log) : IConsumer<BookingConfirmed>
    {
        public async Task Consume(ConsumeContext<BookingConfirmed> ctx)
        {
            var eventId = ctx.MessageId ?? Guid.NewGuid();
            var evt = ctx.Message;
            var contact = await bookings.GetContactAsync(evt.BookingId, ctx.CancellationToken);
            if (contact is null) { log.LogWarning("no contact for booking {Id}", evt.BookingId); return; }

            var idemp = new EmailIdempotencyLog {
                EventId = eventId, EmailType = EmailType.FlightConfirmation,
                BookingId = evt.BookingId, Recipient = contact.Email,
                SentAtUtc = DateTime.UtcNow };
            db.EmailIdempotencyLogs.Add(idemp);
            try { await db.SaveChangesAsync(ctx.CancellationToken); }
            catch (DbUpdateException ex) when (IsUniqueViolation(ex)) {
                log.LogInformation("duplicate {Type} for {Event} skipped", EmailType.FlightConfirmation, eventId);
                return;
            }

            // NOTF-06 branded render
            var rendered = await renderer.RenderAsync(EmailType.FlightConfirmation, new FlightConfirmationModel(
                contact.Name, evt.Pnr, evt.TicketNumber, 0m /* Total resolved via BookingService read if needed */, "GBP"),
                ctx.CancellationToken);

            // NOTF-01 PDF attachment
            var pdf = pdfGen.Generate(new ETicketDocumentModel(

                /* flight details looked up via BookingService if necessary; fallback to placeholders keeps NOTF-01 SLA */

            var envelope = new EmailEnvelope(contact.Email, contact.Name, rendered.Subject,
                rendered.HtmlBody, rendered.PlainTextBody,
                new[] { new EmailAttachment("eticket.pdf", "application/pdf", pdf, "eticket.pdf") });

            if (!result.Success)
