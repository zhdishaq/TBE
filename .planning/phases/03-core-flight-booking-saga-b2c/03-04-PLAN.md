<objective>
Deliver NOTF-01..NOTF-06: a NotificationService that consumes booking lifecycle events from RabbitMQ and dispatches branded transactional emails (with PDF attachments where applicable) within the 60-second SLA, exactly once per (event, email-type) pair.

Concretely:
1. **Email infrastructure**: `IEmailDelivery` + `SendGridEmailDelivery` (SendGrid 9.29.3); `IEmailTemplateRenderer` + `RazorLightEmailTemplateRenderer` (RazorLight 2.3.1) loading cshtml templates from `Templates/` with typed view models.
2. **PDF infrastructure**: `IETicketPdfGenerator` + `QuestPdfETicketGenerator` (QuestPDF 2026.2.4) rendering e-ticket from typed model — Community license at dev time, commercial flagged for production.
3. **Idempotency**: `EmailIdempotencyLog` EF entity keyed by `(EventId Guid, EmailType string)` with unique index; consumers insert-then-send so duplicate event delivery is a no-op (NOTF-05).
4. **Consumers**: `BookingConfirmedConsumer`, `BookingCancelledConsumer`, `TicketIssuedConsumer`, `BookingExpiredConsumer`, `TicketingDeadlineApproachingConsumer`, `WalletLowBalanceConsumer` — each consumes a TBE.Contracts event published in 03-01/03-03, renders the appropriate cshtml template, and dispatches via SendGrid.
5. **Service host**: REMOVE the existing `NotificationService.API/Worker.cs` placeholder; wire MassTransit consumers + RabbitMQ + EF via existing `AddTbeMassTransitWithRabbitMq`. SendGrid + RazorLight + QuestPDF registered in DI.
6. **Templates**: 5 cshtml templates with shared `_Header.cshtml` + `_Footer.cshtml` partials.

Purpose: This plan runs in **Wave 2 in parallel with 03-02** because it shares zero `files_modified` with the payment plan and depends only on the saga contracts published by 03-01 (`BookingConfirmed`, `BookingCancelled`, `TicketIssued`, `BookingExpired` events). It cannot run in Wave 1 because the contracts must exist first.

Output: ~30 new files in NotificationService, 5 unit test classes, removal of Worker.cs placeholder, and a new EF migration adding `EmailIdempotencyLog` table.
</objective>

    ```csharp
    namespace TBE.NotificationService.Application.Consumers;

    using MassTransit;
    using TBE.Contracts.Events;
    using TBE.NotificationService.Application.Email;
    using TBE.NotificationService.Infrastructure.Persistence;

    public sealed class BookingConfirmedConsumer(
        NotificationDbContext db,
        IBookingContactClient bookings,   // typed HttpClient wrapper
        IEmailTemplateRenderer renderer,
        IEmailDelivery delivery,
        ILogger<BookingConfirmedConsumer> log) : IConsumer<BookingConfirmed>
    {
        public async Task Consume(ConsumeContext<BookingConfirmed> ctx)
        {
            var eventId = ctx.MessageId ?? Guid.NewGuid();
            var evt = ctx.Message;
            var contact = await bookings.GetContactAsync(evt.BookingId, ctx.CancellationToken);
            if (contact is null) { log.LogWarning("no contact for booking {Id}", evt.BookingId); return; }

            var idemp = new EmailIdempotencyLog {
                EventId = eventId, EmailType = EmailType.FlightConfirmation,
                BookingId = evt.BookingId, Recipient = contact.Email,
                SentAtUtc = DateTime.UtcNow };
            db.EmailIdempotencyLogs.Add(idemp);
            try { await db.SaveChangesAsync(ctx.CancellationToken); }
            catch (DbUpdateException ex) when (IsUniqueViolation(ex)) {
                log.LogInformation("duplicate {Type} for {Event} skipped", EmailType.FlightConfirmation, eventId);
                return;
            }

            var rendered = await renderer.RenderAsync(EmailType.FlightConfirmation, new FlightConfirmationModel(
                contact.Name, evt.Pnr, evt.ETicketNumber, evt.Total, evt.Currency), ctx.CancellationToken);

            var envelope = new EmailEnvelope(contact.Email, contact.Name, rendered.Subject,
                rendered.HtmlBody, rendered.PlainTextBody, Array.Empty<EmailAttachment>());

            var result = await delivery.SendAsync(envelope, ctx.CancellationToken);
            if (!result.Success)
                throw new InvalidOperationException($"SendGrid failed for {evt.BookingId}: {result.ErrorReason}");

            idemp.ProviderMessageId = result.ProviderMessageId;
            await db.SaveChangesAsync(ctx.CancellationToken);
        }

        private static bool IsUniqueViolation(DbUpdateException ex) =>
            ex.InnerException is Microsoft.Data.SqlClient.SqlException sql &amp;&amp; (sql.Number == 2601 || sql.Number == 2627);
    }
    ```
