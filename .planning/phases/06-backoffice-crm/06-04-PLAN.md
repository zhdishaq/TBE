<objective>
Close CRM-01..05 + COMP-03 by standing up CrmService (event-sourced projections via MassTransit consumers), wiring the Agencies.CreditLimit hard-block into WalletReserveConsumer (D-61), introducing the GDPR erasure flow (D-57) with typed-confirmation + open-saga block + cross-service NULL-PII consumers, and shipping the backoffice portal pages (Customer 360, Agency 360, Upcoming Trips, Erasures Archive, Global Search).

Purpose:
- CrmService is the event-sourcing consumer layer (D-51) — we never reach across service boundaries to read BookingService or PaymentService DBs; we consume integration events and build local projections. InboxState gives idempotent dedup without hand-rolling our own dedup table (deviation from RESEARCH Pattern 5 noted in alternatives).
- D-61 credit-limit enforcement moves overdraft control from a soft alert to a hard block at WalletReserveCommand — the earliest point in the booking saga where failure is recoverable before any BookingSagaState row is stamped.
- D-57 GDPR erasure is the first cross-service NULL-PII workflow — three consumers (CrmService, BookingService, PaymentService) all react to a single `CustomerErasureRequested` event, each clearing their own copy; BookingEvents stay intact per D-49, and the CRM projection layer renders "Anonymised Customer" for post-erasure reads.

Output:
- CrmService fully wired: Program.cs, DbContext, 2 migrations (projections + tombstones), 7 MassTransit consumers, 5 controllers.
- BackofficeService: ErasureController (publishes CustomerErasureRequested, blocks on open saga).
- BookingService: CustomerErasureRequestedConsumer (NULL-PII on BookingSagaState).
- PaymentService: AgencyWallets.CreditLimit migration + extended WalletReserveConsumer + AgencyCreditLimitController.
- Shared contracts: CrmEvents.cs with 4 new events.
- backoffice-web: Customers list + Customer 360 (4 tabs) + Erase dialog + Erasures archive + Agencies list + Agency 360 (7 tabs) + Credit limit dialog + Upcoming Trips + Global Search (cmdk).
- 7 Phase06 integration tests covering projections, credit limit, search, communication log, upcoming trips, erasure, and replay-from-zero.
</objective>

public async Task<ReserveResult> ReserveAsync(Guid walletId, Guid agencyId, decimal amount, Guid correlationId, CancellationToken ct) {
    using var tx = await _db.Database.BeginTransactionAsync(ct);
    // Read balance + limit under UPDLOCK
    var row = await _db.Database.SqlQueryRaw<BalanceLimitRow>(@"
        SELECT COALESCE(SUM(SignedAmount), 0) AS Balance,
               (SELECT CreditLimit FROM payment.AgencyWallets WITH (UPDLOCK, HOLDLOCK) WHERE AgencyId = @AgencyId) AS CreditLimit
        FROM payment.WalletTransactions WITH (UPDLOCK, HOLDLOCK)
        WHERE WalletId = @WalletId",
        new SqlParameter("@AgencyId", agencyId), new SqlParameter("@WalletId", walletId)
    ).FirstAsync(ct);
    var available = row.Balance + row.CreditLimit;
    if (available < amount) {
        await tx.RollbackAsync(ct);
        return ReserveResult.Denied(
            code: "wallet-credit-over-limit",
            message: $"Reserve of {amount:0.00} would exceed credit limit (balance {row.Balance:0.00} + creditLimit {row.CreditLimit:0.00} = {available:0.00} available)",
            httpStatus: 402);

    // Append Reserve row

    await tx.CommitAsync(ct);

- `POST /api/backoffice/customers/{id}/erase` as ops-admin body `{ reason: 'Customer requested via email', typedEmail: 'alice@ex.com' }` where the customer's email is indeed `alice@ex.com`:
  - Lookup customer in CRM projection → compare req.typedEmail to CustomerProjection.Email (case-insensitive, trimmed). If mismatch → 400 problem+json `type=/errors/customer-erasure-typed-email-mismatch`.
  - Check for open saga: query BookingService for BookingSagaState rows with CustomerId=req.id AND Status NOT IN ('Confirmed','Cancelled','Failed') → if any, 409 problem+json `type=/errors/customer-erasure-blocked-open-saga` with the booking id.
  - Check for existing tombstone by EmailHash → if exists, 409 problem+json `type=/errors/customer-already-erased` with the original erasure date.
  - Publish `CustomerErasureRequested(requestId=new, customerId=id, emailHash=SHA256(email.Normalize()), requestedBy=actor, reason, at=UtcNow)` via BackofficeDbContext outbox.
  - Return 202 Accepted with `{ requestId }`.
- CRM consumer `CustomerErasureRequestedConsumer`:
  - Upserts a `crm.CustomerErasureTombstones` row (OriginalCustomerId, EmailHash, ErasedAt, ErasedBy, Reason).
  - NULLs `Customers.Email/Name/Phone` and sets `IsErased=true`, `ErasedAt=At`.
  - Publishes `CustomerErased(CustomerId, EmailHash, ErasedAt)` for downstream observers (audit/alerting).
- BookingService consumer `CustomerErasureRequestedConsumer`:
  - Sets `BookingSagaState.CustomerName/Email/Phone/PassportNumber/DateOfBirth = NULL` for all bookings where CustomerId = req.CustomerId.
  - Does NOT modify BookingEvents table (D-49 immutability preserved).
  - Does NOT delete the BookingSagaState row — financial audit preserved.
- After all 3 consumers process: GET `/api/crm/customers/{id}` returns Customer 360 with placeholder "Anonymised Customer" label (the `IsErased=true` flag drives client-side redaction; API still returns the row with NULL PII fields so the booking history tab still shows bookings).

**E. `CrmService.Application/Consumers/CustomerErasureRequestedConsumer.cs`**:
```csharp
public async Task Consume(ConsumeContext<CustomerErasureRequested> ctx) {
    var m = ctx.Message; var ct = ctx.CancellationToken;
    using var tx = await _db.Database.BeginTransactionAsync(ct);
    // Idempotent tombstone insert via UNIQUE on EmailHash
    var exists = await _db.CustomerErasureTombstones.AnyAsync(t => t.EmailHash == m.EmailHash, ct);
    if (exists) {
        _log.LogInformation("Tombstone already exists for {Hash}; skipping", m.EmailHash);
        await tx.CommitAsync(ct);
        return;
    }
    _db.CustomerErasureTombstones.Add(new CustomerErasureTombstoneRow {
        Id = Guid.NewGuid(), OriginalCustomerId = m.CustomerId, EmailHash = m.EmailHash,
        ErasedAt = m.At, ErasedBy = m.RequestedBy, Reason = m.Reason
    });
    // NULL PII on CustomerProjection
    var cust = await _db.Customers.FirstOrDefaultAsync(c => c.Id == m.CustomerId, ct);
    if (cust != null) {
        cust.Email = null; cust.Name = null; cust.Phone = null;
        cust.IsErased = true; cust.ErasedAt = m.At;
    }
    await _db.SaveChangesAsync(ct);
    await ctx.Publish(new CustomerErased(m.CustomerId, m.EmailHash, m.At), ct);
    await tx.CommitAsync(ct);
}
```
