<objective>
Close B2B-06 (wallet top-up via Stripe for agency admin) and B2B-07 (atomic wallet reservation with append-only ledger + low-balance alerts) end-to-end.

Purpose: Give agency admins a first-class wallet-admin surface (balance, transactions, top-up, threshold), prove the append-only ledger holds under concurrent reserves, and ensure low-balance alerts fire exactly once per threshold crossing. This plan does NOT build the wallet reservation path in the saga -- that already lands in Plan 05-02 via `WalletReserveCommand` / `WalletReserved` / `WalletReserveFailed`. This plan owns the **storage**, **top-up**, **monitoring**, and **admin UI** sides of the wallet.

- `AgencyWallet` table + wallet columns migration
- `WalletTopUpService` (caps + idempotent commit)
- `WalletLowBalanceMonitor` BackgroundService + `WalletLowBalanceConsumer`
- `WalletController` D-40 endpoints (top-up intent / threshold / transactions)
- `/admin/wallet` RSC page + Stripe Elements top-up form + transactions table + threshold dialog
- Sitewide low-balance banner (agent-admin only, sessionStorage dismiss)
- `RequestTopUpLink` (non-admin mailto fallback)
- Concurrency + service tests (including the B2B-07 double-spend case under real SQL Server)
</objective>
