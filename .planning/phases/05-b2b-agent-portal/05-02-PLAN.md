<objective>
Ship the B2B booking-on-behalf workflow end-to-end: (a) pricing-service evaluates agency markup + commission via `AgencyMarkupRules`; (b) search results grid renders 4-column NET / Markup / GROSS / Commission dual-pricing; (c) `BookingSagaState` is extended with B2B channel + agency fields + customer contact; (d) `BookingSaga` branches on `PnrCreated` to a wallet-debit path (`WalletReserveCommand`) for B2B, preserving the Stripe path for B2C; (e) `AgentBookingsController` stamps `Channel=B2B` and `AgencyId` from session claims; (f) `/checkout/details` captures customer contact and exposes `AgencyMarkupOverride` for agent-admin only; (g) `/checkout/confirm` renders debit summary or in-place insufficient-funds panel (no Stripe Elements on this route); (h) wallet chip renders server-side initial balance with 30s polling.

Purpose: B2B-03 (search visible to agents), B2B-04 (booking creation with customer contact snapshot + markup), B2B-05 (saga channel branching), and B2B-06 (wallet debit at reserve) are the core value of this phase. Without this plan the portal has a session but cannot quote, price, book, or debit a wallet.
Output: An agent signs in, searches flights, sees NET alongside GROSS + markup + commission, picks an offer, enters customer contact, confirms, and the agency wallet is reserved at PNR creation; success page shows `?booking={id}` only.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent browser -> `/api/bookings` (gateway) | JWT validated by gateway B2BPolicy (Plan 01); controller re-reads `agency_id` claim |
| Client body -> AgentBookingsController | `agency_id`, `Channel`, `AgencyMarkupOverride` are NEVER trusted from body -- server-side only |
| AgencyPriceRequested consumer -> MarkupRulesEngine | Assumes AgencyId came from a trusted upstream (gateway-authenticated controller) |
| BookingSaga B2B branch -> Payment service wallet | MassTransit contract; idempotency key = BookingId ensures no double reserve |
| Checkout/confirm -> balance check | Server-side balance read; UI cannot bypass insufficient-funds by manipulating client state |

    public interface IMarkupRulesEngine
    {
        Task<AgencyPriceQuoted> ApplyMarkupAsync(Guid agencyId, decimal netFare, string? routeClass, string currency, string offerId, Guid correlationId, CancellationToken ct);
    }

    public sealed class MarkupRulesEngine : IMarkupRulesEngine
    {
        private readonly IPricingDbContext _db;
        private readonly ILogger<MarkupRulesEngine> _log;

        public async Task<AgencyPriceQuoted> ApplyMarkupAsync(Guid agencyId, decimal netFare, string? routeClass, string currency, string offerId, Guid correlationId, CancellationToken ct)
        {
            if (netFare < 0) throw new ArgumentOutOfRangeException(nameof(netFare));
            // D-36: override ?? base
            var rules = await _db.AgencyMarkupRules
                .Where(r => r.AgencyId == agencyId && r.IsActive)
                .ToListAsync(ct);
            var rule = rules.FirstOrDefault(r => r.RouteClass == routeClass)
                    ?? rules.FirstOrDefault(r => r.RouteClass == null);
            if (rule is null)
            {
                _log.LogWarning("No active markup rule for agency {AgencyId}; zero-markup quote", agencyId);
                return new AgencyPriceQuoted(correlationId, agencyId, offerId, netFare, 0m, netFare, 0m, currency);
            }
            var markup = rule.FlatAmount + netFare * rule.PercentOfNet;
            var gross = netFare + markup;
            var commission = markup;   // v1: commission = markup; settlement deferred per D-41
            return new AgencyPriceQuoted(correlationId, agencyId, offerId, netFare, markup, gross, commission, currency);
        }
    }
