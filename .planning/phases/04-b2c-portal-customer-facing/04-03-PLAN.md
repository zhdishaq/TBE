<objective>
Ship the hotel search + detail + booking surfaces for the B2C portal AND the server-side hotel voucher email pipeline (NOTF-02 primary owner). Hotel flows reuse the same checkout/payment/processing/success pages built in 04-02 by pushing a normalized "offer" into the same basket interface, so this plan adds **zero** new checkout pages. It adds: search form + results + detail + room picker UI, the HotelBookingConfirmed event + NotificationService consumer + RazorLight template + QuestPDF document, a BookingService HotelBookingsController + HotelBookingSagaState, and a dashboard voucher download pass-through.

Purpose: Close HOTB-01..05 + NOTF-02, and lay the shared hotel-booking primitives the Trip Builder (04-04) will compose with flights.
Output: A customer can book a hotel end-to-end, receives a voucher email with PDF attachment within 60s, and can re-download the voucher from the dashboard.
</objective>
